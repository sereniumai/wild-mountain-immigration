## Per-page on-page standard (apply to EVERY page)
1. **Primary keyword in the URL slug** (only change if low-risk; otherwise keep + add redirects later).
2. **<title>** ~50-60 chars incl. brand suffix, primary KW front-loaded.
3. **meta description** <=155 chars, primary KW + a benefit + soft CTA.
4. **H1** contains the primary KW (natural).
5. **H1 subtext / lede** repeats the KW once + the answer-intent.
6. **First H2** uses the KW or a close variant; **one more heading** uses a variant/LSI.
7. **KeyTakeaways** = a direct, snippet/AI-ready answer to the core query (40-55 words), KW in the first sentence.
8. **Body**: natural KW density (~0.5-1.2%), strong **LSI/entities** (program names, NOC, IRCC, CRS, etc.).
9. **FAQs**: 5-8, mapped to real "People Also Ask" / question keywords; emitted as FAQPage schema.
10. **Schema**: correct Article/Service/FAQ/Breadcrumb (+ LocalBusiness/Person sitewide), accurate only.
11. **Internal links**: if a primary KW changes, update inbound anchor text across the site to match.

## Slug decision
Slugs were evaluated against each page's best keyword and left unchanged: the keyword is already in the full hierarchical path (e.g. /express-entry/draws = "express entry draws", /pnp/nova-scotia/physician), acronym slugs are themselves high-volume (/work-permits/pgwp), and conventional slugs (/about, /fees, /services, /contact) should stay conventional. No slug change was a net SEO win; renaming would break links and read as spammy.
